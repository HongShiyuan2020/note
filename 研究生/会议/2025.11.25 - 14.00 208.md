## 对象与状态
1. 探究凸透镜的成像规律
	1. 仪器：光具座、直尺、F光源、焦距已知的凸透镜、光屏
	2. 状态：
		- 光屏上成像的清晰程度（光屏）
		- 成像的倒正（光屏）
		- 成像的大小（光屏、直尺）
		- 成像为实像或虚像（光屏）
		- 像距与物距比例（光具座刻度尺）
		- 光具座上三者是否共轴（光具座）
		- F 光源发光状态（F 光源）
		- 透镜位置是否垂直光轴（凸透镜、光具座）
2. 探究平面镜成像规律
	1. 仪器：深色透明玻璃板、玻璃板底座、白纸、铅笔、两个塑料小蜡烛、、手持光屏
	2. 状态：
		- 像的位置（手持光屏 / 玻璃板观察） 
		- 像的大小与物体对比（手持光屏、白纸、铅笔）
		- 像的左右关系（玻璃板、蜡烛）
		- 像与物的距离比较（白纸、尺可选）
		- 两个蜡烛是否对称（蜡烛、白纸）
		- 玻璃板是否竖直摆放（玻璃板底座）
		- 光线标记是否准确（白纸、铅笔）
3. 探究光的反射定律：
	1. 仪器：光的反射仪器（一个巨大的量角器）、铅笔、红色激光笔、绿色激光笔
	2. 状态：
		- 入射光线位置（激光笔、量角器）
		- 反射光线位置（激光笔、量角器）
		- 法线位置是否画准（量角器、铅笔）
		- 入射角的读数（量角器）
		- 反射角的读数（量角器）
		- 镜面是否与基线重合（量角器）
		- 红光与绿光的反射路径差异（激光笔、量角器）
4. 用温度计测量温度：
	1. 仪器：三个烧杯、一只量程包括0-100摄氏度的温度计、热水、凉水些许
	2. 状态：
		- 温度计示数（温度计） 
		- 温度计玻璃泡是否完全浸入液体（温度计、烧杯）
		- 液体温度变化趋势（温度计、烧杯）
		- 温度计是否接触到杯壁或杯底（温度计、烧杯）
		- 热水与凉水的混合均匀程度（烧杯）
5. 探究水沸腾的规律：
	1. 仪器：加热器、水、热水壶、铁架台、一只量程包括0-100摄氏度的温度计、秒表
	2. 状态：
		- 温度计示数随时间的变化（温度计）
		- 是否出现大量气泡（水壶、加热器）  
		- 气泡上升方式（水壶）
		- 加热时间记录（秒表）
		- 温度是否在沸点附近稳定（温度计）
		- 温度计位置是否悬空不接触壶底（温度计、铁架台）
6. 探究欧姆定律：
	1. 仪器：电压表、电流表、滑动变阻器、三个不同规格的电阻、导线若干、开关、电池、
	2. 状态：
		- 电流表读数（电流表）
		- 电压表读数（电压表）
		- 滑动变阻器滑片位置（滑动变阻器）
		- 电阻两端电压变化（电压表）
		- 电流随电压变化的趋势（电流表、电压表）
		- 电路是否接通（开关）
		- 是否出现仪表指针反向（电流表、电压表）
7. 探究并联电路电流与电压的特点：
	1. 仪器：三个电压表、三个电流表、两个不同规格的灯泡、电池、开关、导线若干
	2. 状态：
		- 主电路电流（电流表 A1） 
		- 各支路电流（电流表 A2、A3）
		- 各灯泡两端电压（电压表 V1、V2）
		- 电池两端电压（电压表 V0）
		- 灯泡亮度（灯泡）
		- 两灯泡是否串联（导线、开关）
8. 用电流表测电流：
	1. 仪器：电流表、灯泡、电池、开关、导线若干
	2. 状态：
		- 电流表读数（电流表）
		- 电流表接法是否串联正确（电流表、导线）
		- 电流方向是否正确（电流表）
		- 灯泡亮度（灯泡）
		- 电路是否接通（开关）
9. 用电压表测电压：
	1. 仪器：电压表、灯泡、电池、开关、导线若干
	2. 状态：
		- 电压表读数（电压表）
		- 电压表是否正确并联（电压表、导线）
		- 被测两点是否接触牢固（导线、电压表）
		- 灯泡亮度（灯泡）
		- 电路是否接通（开关）

## KG边

- 属性边（连接状态节点与对象节点）
```json
{
  "from": "state_000001",
  "to": "object_001",
  "type": "attr",
  "relation": "借由此测量所得"
}
```

- 迁移边（连接当前时刻状态节点和上一时刻状态节点的边）
```json
{
  "from": "state_000001",
  "to": "state_000005",
  "type": "trans",
  "relation": "图像中的新状态",
  "reason": "状态改变的原因"
}
```

- 知识关联边（连接状态或对象和知识）
```json
{
  "from": "pt_001",
  "to": "kng_001",
  "type": "knowledge_edge",
  "relation": "explained_by",
  "reason": ""
}
```
- 知识边（连接知识节点与知识节点）
```json
{
  "from": "state_0007_current",
  "to": "kn_ohm_01",
  "type": "konwledge_link_edge",
  "relation": "explained_by",
  "reason": ""
}
```
- 增强边（连接增强节点和被增强对象的边，例如：凸透镜实验中关键状态为成像模糊，该状态和对象节点凸透镜、光源和光屏连接，增强时基于凸透镜添加了相反的概念凹透镜节点，将其与凸透镜节点连接）
```json
{
  "from": "state_0007_current",
  "to": "kn_ohm_01",
  "type": "konwledge_link_edge",
  "relation": "explained_by",
  "reason": ""
}
```

## 如何构建KG

KG中的节点和边可以分为两大类：
1. 静态节点和静态边：节点、边的属性和连接关系不需要图像就可以确定。
	1. 知识节点
	2. 知识关系边
2. 动态节点和动态边：节点、边的属性和连接关系需要图像才可以确定。
	1. 对象节点
	2. 属性边（存在固定的匹配模式）
	3. 状态节点
	4. 对象-知识边 （存在固定的匹配模式）
	5. 迁移边 （存在固定的匹配模式）
	6. 增强节点
	7. 增强边 （存在固定的匹配模式）

## 如何提取核心状态节点

1. 获取每个状态节点的固定深度的周边节点描述
2. 基于上面的信息进行关键度排序

关键节点表示了图像描述的关键状态，是学生最容产生疑惑的状态，通常表示为KG中的状态节点。
**困惑度如何定义**。

### “三源融合”方法：

#### **（A）基于图像——多模态模型感知不确定性**

使用一个视觉大模型（如 GPT-4o / Qwen-VL2）对图像进行“多次抽样问答”，  
找到**模型最不确定的状态**。

示例：

- 多次问：图中电流表读数是多少？→ 方差大 → 说明该节点存在视觉不确定性
- 问：这是否是串联？→ 若模型反复纠结 → 结构关系存在不清晰点

**这些节点是“学生容易犯错”的强信号”。**

---

#### **（B）基于知识图谱结构——高影响力节点**

计算 KG 中状态节点的中心性（degree, betweenness），高中心性节点意味着：

- 被多个知识点解释
- 与多个仪器有关
- 是实验结论的重要部分

这些节点**高度适合出题**。

---

#### **（C）基于知识点——查找知识冲突点**

利用知识点之间的知识边：

- contradicts（矛盾）
- requires（前置知识）
- related_to（容易混淆）

查找：

- 哪些状态同时依赖多个互相关联但容易混淆的知识点  
    ➡ 这类状态才是学生最容易搞错的。

---

#### **最终评分（可直接用）**

对每个状态节点 s 计算：

```
confusion_score(s) = 
      α * vision_uncertainty(s) 
    + β * knowledge_dependency(s)
    + γ * graph_centrality(s)
```

取 confusion_score 前 N 名作为关键节点。

## 如何生成多样化问题

### 第一层：抽象题型模板（题型元结构）

- 定量读取题
- 比较变化题（如调节滑片前后）
- 错误诊断题（常考）
- 结构判断题
- 知识溯源题（问为什么）
- 预测结果题
- 动作推断题（“学生做了什么导致电流变化？”）

### 第二层：槽位填充（由 KG 提供）

例如一个模板：

**“某状态__TARGET_STATE__与其关联的仪器__TARGET_INSTRUMENT__之间是什么关系？”**

模型根据 KG 自动填入：
- TARGET_STATE
- TARGET_INSTRUMENT
- 他们之间的 relation
- reason
- 时间片信息

### 第三层：生成扩展（用 LLM 转写）

对模板生成的问题，再让模型进行“表达变换”：

- 改写风格
- 使用不同提问方式
- 拆成两问
- 添加真实教学语气

例如：

- “请判断” → “你认为是否正确？”
- “读数是多少” → “大致是几安？”
- “根据图像” → “结合图示判断”
### 增加增强节点

基于提取到的核心状态节点相关的节点有大模型联想加入

- 语义相近节点
- 语义相反节点
- 随机场景描述节点
### 题目类型

- 读取型
- 推理型
- 变化型
- 错误型
- 结构型
- 应用型
